About Us

The Platypus Security Team has been working on specific security solutions and products for the past 10 years as a public benefit and according to the interests of the team members. We have provided various free solutions so that all people can access the free internet.
This team now intends to commercialize one of the solutions (Platypus), by attracting investors or selling the original source of this product or selling versions of this product in the number of at least 200 (to famous service providers).
Monitoring of censorship events around the world and reviews of publicly provided data by top organizations such as the Open Observatory of Network Interference (OONI) and Internet Outage Detection and Analysis (IODA) show that in many countries There is severe filtering, including in Iran, China and Russia, which have the highest level of filtering, and the possibility of using filter breakers is extremely limited.

On the other hand, a very large market has been formed in the world, which has led to large income generation with very high efficiency and ROI, which has satisfied investors.

Strong Kernel

> Platypus is currently installed and launched on the kernel of the powerful FreeBSD operating system. but it is easily possible to develop it on other operating systems as well.

  • FreeBSD is an extremely clean and predictable OS.
  • FreeBSD can act as really a good alternative to traditional UNIX platforms
  • FreeBSD is a monolithic kernel, and its primary interest is security
  • FreeBSD is stable

Of course, there are several reasons why we choose FreeBSD. Several points, are self-explanatory to use FreeBSD. It is highly customizable, and you can customize it as per your requirement.

Flexible UI

One of the attractions of Platypus, which makes for a good experience in using this product, is having a simple yet powerful UI.

  • It has WEB management interface
  • It has GUI management interface
  • It has CLI management interface
  • It has central intergrated management interface
  • It has simple client interface (Currently in Windows and in the next development of other OS including Android, Linux, Mac and iPhone )

Various Protocols

Platypus offers a proprietary protocol for its VPN that sets it apart from others. It is not possible to identify and block this protocol by security tools (including firewall). Of course, the explanation and performance of this solution as capital is only available to us and I emphasize that this solution does not exist in any other product in the world.
But Platypus also support other common VPNs, and in future developments, it is even possible to add newer ones.

  • Platypus Special Protocol (PSP3)
  • IPSec (Internet Protocol Security) IKEv1/2
  • SSL/TLS (Secure Socket Layer / Transport Layer Security protocols)
  • L2TP (Layer 2 Tunneling Protocol)
  • PPTP (Point–to–Point Tunneling Protocol)
  • OpenVPN & Wireguard if you want

User Management

Users and customers are your most important capital, you can easily define unlimited users (Domain, Local, LDAP) in this product and apply many restrictions and settings on them.

  • User bandwidth control limits
  • Quota limits and Time limits of use
  • Price limits and Credit parameters
  • Ability to use Hardware Tokens
  • Complete logging of visited sites and user communications
  • And more than 50 types of user restrictions

VPNaaS (Cloud/Hardware/VM)

Platypus can be installed and presented in different ways.

  • Cloud => Can be installed in Azure, AWS and other infrastructure
  • Hardware => Setup on hardware servers and network appliances
  • Virtual Machine => Installation as a virtual machine on ESXi, KVM, VirtualBox and others

Super Security

In the Platypus product, the security of the infrastructure and the security of users' access to the secure Internet have been our biggest and most important concerns.

We have taken important steps for security, from the implementation of a dedicated FIREWALL to the installation of various security modules so that you do not need additional security equipment. We are checking and making this solution more secure every day.

  • Defining security policies at different levels for secure firewalling
  • Security modules to control DoS attacks and prevent DDoS attacks
  • Controlling access to unsafe and dangerous sites, including phishing
  • Geo-IP Controls and IP Reputations
  • Application/Identity Awareness and Port Enforcement
  • And many other security features
